Elasticsearch (ES) 是一个分布式的搜索和分析引擎。它被广泛应用于各种场景，如日志分析、全文搜索、指标监控等。Elasticsearch 的核心是一个基于 Apache Lucene 的搜索引擎。简单来说，它能够快速存储、搜索和分析大量数据，类似于数据库，但其主要功能是搜索和分析数据。
如果直接跟es交互使用的是restful接口的形式。Elasticsearch 提供了一套 RESTful API 来进行交互。通过这些 API，你可以实现数据的索引、搜索、更新和删除等操作。RESTful 接口使用 HTTP 请求方法（如 GET、POST、PUT、DELETE）与 Elasticsearch 进行通信，并返回 JSON 格式的数据。

查询很厉害:
term和terms查询不分词 但是terms用于一个字段包含多个词的时候(相当于一次可以指定多个,类比可以是关系型中的in)
查询的时候关键字不会被分词的情况下可以搜到被搜索的结果中分词结果有对应关键字的

查询结果中会有一个分数,匹配度越高分数就越高,展示就会越靠前

match查询属于高级查询形式,它会根据你的日期的不同使用不同的查询方式 match查询底层其实还是term查询
match特定的内容的时候会将关键字分词之后再去查询,再去分词完成的内容库中去匹配内容

match多条件查询的时候中间是空格分隔 之后指定operatpr

multi_match是对多个条件进行检索,可以通过一个条件在多种field中查询

fuzzy还能模糊查询 进行大概匹配 但是不太文稳定

wildcard是通配查询 和MySQL中的like是一个套路 *是任意数量 ?是占位一个字符

range范围查询 带e就是带等于号

regexp是正则查询 按照自己的正则表达式 列斯 180[0-9]{8}

其中fuzzy prefix wildcard regexp查询效率相对较低 要求效率的时候尽量不要用
